### 基础初始化

让我们首先观察最基本的插件初始化代码：

```rust
#[tokio::main]
async fn main() {
    let (plugin, Initialize { .. }) = plugin!();
}
```

在这段代码中：

<Card title="plugin! macro" className="mb-4">
`plugin!` 宏封装了插件的初始化逻辑，它会创建一个新的插件实例并与框架进行握手。
</Card>

<Card title="返回值" className="mb-4">
返回一个元组，其中第一个元素是插件实例，第二个元素是初始化结果 `Initialize` 结构体。
</Card>

<Card title="泛型参数">
`plugin!` 宏的泛型参数表示插件的配置类型，如果省略，则默认为 `()`，即空配置。
</Card>

### Initialize 结构体

初始化结果封装在 `Initialize` 结构体中，它的定义如下：

```rust
pub struct Initialize<C> {
    pub config:    C,
    pub data_path: String,
    pub id:        String,
}
```

<Callout>

这个结构体包含了两个重要字段：

- `config` 字段表示插件的配置信息，可以为任何可反序列化类型
- `data_path` 字段表示插件的数据路径，我们约定插件的所有数据都存储在该路径下
- `id` 字段表示插件的唯一标识符，用于在框架中区分不同的插件实例。

</Callout>

### 自定义插件配置

通过为 `plugin!` 宏指定泛型参数，我们可以定义插件专属的配置类型：

```rust
struct Config {
  admins: Vec<String>,
}

#[tokio::main]
async fn main() {
    let (
        plugin,
        Initialize { config, .. }
    ) = plugin!(Config);

    // 现在 config 是 Config 类型，可以访问 admins 字段
    for admin in &config.admins {
        log::info!("管理员: {admin}");
    }
}
```

### 运行插件

初始化完成后，我们需要启动插件服务并处理退出信号：

```rust
tokio::select! {
    _ = plugin.run().join_all() => {
        // 插件出现内部错误则会执行该分支
    }
    _ = tokio::signal::ctrl_c() => {
        // 插件停止
        // 主要用于优雅地退出插件
        // 比方说关闭数据库连接，释放资源等。
    }
}
```

<Callout type="warn" icon={<Terminal />}>

当插件退出时，框架内部会直接丢弃插件实例，析构时插件会被 kill。

因此在 `tokio::signal::ctrl_c()` 分支中**无法使用**任何需要和框架通信的方法/函数，例如 `log::*`。

</Callout>

### 添加额外任务

如果你的插件依赖于其他关键任务，可以将其添加到 `select` 宏中：

```rust
tokio::select! {
    _ = connection_task => {
        log::error!("Connection manager task exited unexpectedly.");
    }
    _ = plugin.run().join_all() => {}
    _ = tokio::signal::ctrl_c() => {
        log::info!("Shutting down adapter..."); // 这里的 log 并不会执行
    }
}
```

`Plugin` 结构体是一个高级封装，适合一般插件开发者使用。但有时我们需要更精细的控制，因此让我们深入了解其内部实现。

### Plugin 结构体剖析

`Plugin` 结构体的定义如下：

```rust
pub struct Plugin {
    peer:   Peer,
    server: Server,
    router: Router,
}
```

<Card title="peer" icon={<ArrowDown className="h-4 w-4" />} className="mb-4">
  插件的对等节点，用于与框架进行通信
</Card>

<Card title="server" icon={<Workflow className="h-4 w-4" />} className="mb-4">
  插件的服务端，用于处理请求和事件，同时还被用于发送请求并接收响应
</Card>

<Card title="router" icon={<Puzzle className="h-4 w-4" />}>
  插件的路由表，用于管理插件的路由规则

### Plugin::new 方法解析

让我们拆解 `Plugin::new` 方法，了解其内部实现：

```rust
pub async fn new<Config>(version: &str, name: &str) -> (Self, Initialize<Config>)
where
    Config: for<'de> Deserialize<'de>,
{
    // 处理命令行选项，如 --version
    if handle_options(version, name) {
        process::exit(0);
    }
    // 创建对等节点、服务端和路由表
    let peer = Peer::new();
    let server = Server::new();
    let router = Router::new();
    // 从对等节点创建帧序列
    let mut framed = crate::transport::util::framed(peer);

    // 执行初始化循环，等待框架发送初始化消息
    let init = loop {
        let Some(msg) = <FramedPeer as StreamExt>::next(&mut framed).await else {
            break Err(PluginInitError::ConnectionClosed);
        };
        if let Ok(msg) = msg {
            let is_init = msg.path.as_ref().is_some_and(|p| p == Initialize::<Config>::path());
            if is_init {
                let config = msg.payload::<Initialize<Config>>();
                break config.map_err(PluginInitError::ConfigDeserializeError);
            }
        }
    };

    // 处理初始化结果
    let init = match init {
        Ok(init) => init,
        Err(err) => {
            // 发送错误信息给框架
            framed
                .send(
                    DataPack::builder()
                        .path(Initialize::<Config>::path())
                        .build_with_payload(InitializeResult::Err(err)),
                )
                .await.ok();
            // 等待退出信号
            tokio::signal::ctrl_c().await.ok();
            // 退出进程
            process::exit(1);
        }
    };

    // 初始化日志系统
    init_log(server.client().sink());

    // 发送初始化成功响应给框架
    server
        .client()
        .send(
            RequestDataPack::default()
                .path(Initialize::<Config>::path())
                .payload(InitializeResult::Ok(())),
        )
        .unwrap_or_else(|_| panic!("Failed to send initialization response: [{name}]"));

    // 用不到帧序列了，我们换回对等节点
    let peer = framed.into_inner();
    
    // 返回实例
    (
        Self {
            peer,
            server,
            router,
        },
        init,
    )
}
```

<Callout type="info" icon={<Database />}>

初始化过程的关键步骤：

1. 处理命令行参数（如 `--version`）
2. 创建通信组件（对等节点、服务端、路由表）
3. 等待并解析框架的初始化消息
4. 如果初始化失败，向框架报告错误并退出
5. 初始化日志系统
6. 向框架发送初始化成功响应
7. 返回插件实例和初始化结果

</Callout>

### 运行原理

`Plugin::run` 方法的实现相对简洁：

```rust
pub fn run(self) -> JoinSet<Result<(), ServerError>> {
    let Self {
        peer,
        server,
        router,
    } = self;
    // 分割对等节点为写入和读取部分
    let (write, read) = peer.split();

    // 将路由表作为服务，启动服务端
    server.service(router).serve(write, read)
}
```

这个方法将插件的核心组件拆分并组装为服务，使其能够处理请求和发送响应。
